# 安全策略

感谢你帮助提升 `deepseek_chat` 的安全性。

## 漏洞上报方式

请优先使用以下方式之一上报漏洞：

- GitHub Security（如仓库已开启）：`Security` 标签页提交私密报告
- 如果暂未开启 Security 报告，请在仓库中创建 Issue，并避免公开敏感细节：
  - https://github.com/EagleFandel/deepseek_chat/issues

## 披露原则

为保护用户和部署环境，请遵循：

- 不要在公开 issue、PR、评论中粘贴 API Key、Token、Cookie、数据库凭据。
- 不要公开可直接复现且尚未修复的利用细节。
- 可先提供影响范围与复现思路，等待维护者沟通后再补充细节。

## 响应时效（SLA）

- 我们目标在 **72 小时内**给出首次响应。
- 会在确认问题后提供修复状态更新与预估时间。

## 支持版本

当前以 `main` 分支的最新代码为优先支持对象。

- 若问题仅存在于历史版本，可能会建议升级到最新版本验证。
- 关键安全问题会优先在 `main` 修复，并根据影响评估是否回补。

## 安全最佳实践

- 通过 `.env.local` / 平台环境变量管理密钥。
- 提交代码前检查是否误提交敏感信息。
- 在日志中打码处理用户数据与凭据。
